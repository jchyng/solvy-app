Read `design-system/INDEX.md` in the project root — it contains all essential tokens (colors, typography, spacing) and tells you which file to open next.
For detailed guidelines (voice & tone, component patterns, animation rules), read `design-system/README.md`.
For actual component implementations, explore `design-system/ui_kits/`. The `preview/` folder is visual-only HTML; skip it.

If creating visual artifacts (mocks, throwaway prototypes), produce static HTML files for the user to view.
If working on production code, apply the design rules as an expert in this brand.
If invoked without guidance, ask what the user wants to build or design, then act as an expert designer who outputs HTML artifacts or production code depending on the need.
