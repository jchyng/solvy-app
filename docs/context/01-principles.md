# 01. 제품 원칙 · 금기 사항

> **이 문서는 모든 기능 설계·구현 결정에서 참조됩니다.** 새 기능을 추가하거나 기존 기능을 바꾸기 전 반드시 이 원칙들과 충돌하지 않는지 확인하세요.

---

## 5대 제품 원칙

1. **"풀이를 자산으로 만든다"**
   즐겨찾기·목록·이름 지정이 1급 시민 기능. 노트로 보관하지 않으면 Solvy의 가치가 드러나지 않음.

2. **"답이 아니라 풀이법을 판다"**
   출제 의도·활용 개념·실전 팁이 없는 답은 출력하지 않음.

3. **"꼬리 질문으로 깊이를 확장한다"**
   다른 풀이법·변형 문제·선행 개념은 본문에 쏟아붓지 말고 칩으로 유도.

4. **"틀렸을 때 틀렸다고 말한다"**
   confidence 기반 UX, 인식 결과 확인 단계, "확실하지 않습니다" 명시.

5. **"학생의 루틴을 존중한다"**
   기본값 변경 금지, 팝업 최소화, X 버튼 필수, 광고 없음.

---

## ❌ 절대 하지 말 것 (콴다 반면교사)

- 강제 광고, 인터랙티브 광고, 게임 광고
- 무료로 시작한 기능을 유료로 전환
- 업데이트로 기본값 강제 변경 (사용자 동선 보호)
- AI 정확도 불안정한데 AI-only로 밀기
- 팝업에 X 버튼 없이 강제 노출
- 광고 차단 앱 감지 과민 반응
- 템플릿성 고객 대응
- OCR 결과를 검증 없이 바로 사용 (confidence 낮으면 확인 단계 필수)
- 한 번의 분석에 다른 풀이법·변형 문제·선행 개념을 다 쏟아붓기 (꼬리 질문으로)
- 노트·목록 데이터를 학생 본인 동의 없이 이전·삭제
- **학부모 관련 UI·API·데이터 모델 재도입** (Phase 1 범위 이탈)
- 프론트엔드에 API 키 노출
- 사용자 노트 데이터를 학습에 사용

---

## ✅ 반드시 할 것

- AI가 불확실할 때 "확실하지 않습니다" 명시
- 인식 결과를 사용자가 수정할 수 있게
- 모든 문제 풀이가 대화방으로 저장되고, 언제든 돌아올 수 있게
- 즐겨찾기·목록·이름 지정은 **단일 탭·단일 입력**으로 마찰 없이
- 꼬리 질문은 **실제 제공 가능할 때만** 노출 (빈 칩 금지)
- 모든 AI 호출을 `usage_events`에 기록
- 역할별 모델 매핑을 config로 관리
- 폴백 체인 구현
- Anthropic 프롬프트 캐싱 활용 (특히 `ai.chat`)
- 베타 참여자 혜택 + 노트 영구 보존 약속 문서화

---

## 의사결정 체크 질문

새 기능·UI를 추가하기 전 다음을 스스로 물어보세요:

1. 이게 "풀이를 자산으로 만드는" 축에 기여하는가?
2. 학생의 기존 동선을 강제로 바꾸는가? (→ 바꾸면 안 됨)
3. 무료로 제공하다가 나중에 유료화할 여지가 있는가? (→ 있으면 처음부터 유료로 출시)
4. AI가 틀릴 수 있는 영역인데 확신 있게 보여주는가? (→ 신뢰도 표시 필요)
5. 이 기능이 Phase 1 범위인가, Phase 2+ 인가? (→ 주차별 스프린트 문서의 "건드리지 말 것" 확인)

하나라도 걸리면 구현 전에 재검토.
