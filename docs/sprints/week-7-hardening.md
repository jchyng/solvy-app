## 보안 체크리스트

- [ ] API 키 전 구간 서버사이드 유지 (프론트 노출 없음 재검증)
- [ ] CORS 설정 검토
- [ ] 이미지 업로드 용량 · 포맷 제한
- [ ] Rate limit 최종 조정 (역할별 세분화)
- [ ] SQL injection · XSS 방어 (ORM/ESC 사용, 수동 쿼리 최소화)
- [ ] **사용자 간 데이터 접근 권한 교차 검증** — 타인 `conversation_id`, `folder_id`, `problem_session_id` 직접 접근 차단
  - 자동화된 테스트 케이스 작성: 사용자 A의 JWT로 사용자 B의 리소스 접근 시 403
- [ ] 환경변수 · 시크릿 누설 스캔 (gitleaks CI 통합)

## 테스트 체크리스트

- [ ] 벤치마크 문제 100개 회귀 테스트 자동화 (분석 품질)
- [ ] 전체 플로우 e2e 테스트 (Playwright):
  1. 로그인
  2. 업로드 → 분석
  3. 꼬리 질문 탭 → 대화 이어가기
  4. ★ 즐겨찾기
  5. 새 폴더 생성
  6. 폴더에 대화 추가
  7. 이름 변경
  8. 노트 홈에서 재진입
- [ ] 부하 테스트 — 동시 100 사용자 시나리오 (k6)
- [ ] 프로바이더 장애 주입 테스트 — 폴백 체인 작동 확인
- [ ] 대화 히스토리 길이 한계 테스트 (20턴+)
