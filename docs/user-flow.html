공개 엔드포인트
POST /api/v1/waitlist는 JWT 인증 없이 접근 가능한 유일한 공개 뮤테이션 엔드포인트. 이메일 소문자 정규화·앞뒤 공백 trim·중복 멱등 처리를 서버에서 수행한다.

state machine

업로드 플로우

problemStore.ts가 관리하는 6-상태 머신. 2초 간격 폴링으로 백엔드 분석 완료를 감지한다.

%%{init: {'theme':'base'}}%% stateDiagram-v2 direction LR [*] --> idle idle --> uploading : 파일 선택\nPOST /problems uploading --> polling : 세션 ID 수신 uploading --> error : HTTP 오류 polling --> polling : 2초 간격 poll\nGET /problems/:id/status polling --> confirming : status = confirming polling --> done : status = done\n+ conversationId polling --> error : status = error\n또는 30회 초과(60s) confirming --> polling : 텍스트 수정 후 확인\nPOST /problems/:id/confirm done --> [*] : navigate /chat/:id error --> idle : 재시도 (reset)
상태별 렌더링 컴포넌트
  • idle → UploadView — 카메라 버튼 · 앨범 버튼 · 드래그&드롭 영역
  • uploading / polling → LoadingView — 3단계 진행 표시 (이미지 저장 중 → 문제 인식 중 → 분석 중)
  • confirming → ConfirmView — OCR 인식 텍스트 편집 textarea + "분석 시작" 버튼
  • error → LoadingView (error variant) — 오류 메시지 + 재시도 안내
OCR Confidence 정책
백엔드가 confidence 임계값 미만 결과를 반환하면 status = "confirming"으로 응답한다. 사용자가 텍스트를 수정·확인한 뒤에만 분석이 재개된다. (POST /problems/:id/confirm)

분석 & 채팅 플로우

ChatPage의 전체 분기 구조. 후속 질문 칩 유형에 따라 일반 채팅과 유사 문제 생성 경로로 갈라진다.

SSE 스트리밍 이벤트 규약
  • token — streamingContent에 누적 → 말풍선 실시간 갱신
  • done — 최종 MessageResponse 커밋 + follow_up_questions 설정
  • error — 오류 메시지 표시 + 동일 idempotency_key로 재시도 버튼 노출

interaction reference

ChatPage 인터랙션 카드

ChatPage에서 사용자가 취할 수 있는 모든 액션과 각 액션이 호출하는 API.

⭐

즐겨찾기 토글

PATCH /conversations/:id { is_favorite: boolean } — 별 아이콘 클릭 시 즐겨찾기 상태를 토글한다. 활성 시 골든 앰버(#F5A623)로 채워진다.

✏️

이름 변경

RenameModal 열기 → 텍스트 입력 → PATCH /conversations/:id { title }. 최대 40자.

📁

목록에 추가

AddToFolderSheet 열기 — GET /folders로 폴더 목록 불러와 다중 선택. 체크 시 POST /folders/:id/conversations, 체크 해제 시 DELETE /folders/:id/conversations/:convId.

💬

후속 질문 칩

마지막 assistant 메시지 아래 표시. 스트리밍 중에는 숨겨짐. 탭 시 칩 레이블을 메시지로 즉시 전송.

판별식이 뭔가요? 다른 풀이법 알려줘 비슷한 문제 만들어줘
🎯

유사 문제 난이도 선택

칩 레이블에 "유사" 또는 "비슷한"이 포함된 경우, 난이도 선택 UI를 먼저 노출한 뒤 POST /conversations/:id/similar-problem { difficulty }를 호출한다.

같은 난이도 한 단계 위 한 단계 아래
👍

피드백 버튼 ("이 풀이가 도움됐나요?")

분석 완료 후 마지막 assistant 메시지 아래 표시. 클릭 시 posthog.capture('analysis_helpful', { helpful, conversationId }). 클릭 후 "감사합니다" 텍스트로 교체되며 재클릭 불가.

🔄

다시 시도 버튼

SSE error 이벤트 수신 시 노출. 동일한 idempotency_key를 재사용해 서버 중복 처리를 방지한다.

↗️

이 문제로 새 대화 시작

SimilarProblemCard 하단 버튼. POST /problems/from-text → 상태 폴링 → /chat/:newId로 navigate.

api reference

API 엔드포인트 목록

POST /waitlist를 제외한 모든 API 호출은 services/api.ts의 authHeaders()를 통해 Authorization: Bearer {token} 헤더를 포함한다.

Method Endpoint 설명 호출 위치
Waitlist — 공개 (인증 불필요)
POST /waitlist 베타 대기열 이메일 등록 (멱등·소문자 정규화) LandingPage 이메일 폼
Upload — problemStore
POST /problems 이미지 업로드 (FormData) UploadView → upload()
GET /problems/:id/status 상태 폴링 (2초 간격) problemStore._startPolling()
GET /problems/:id 세션 데이터 전체 조회 confirming, done 전환 시
POST /problems/:id/confirm OCR 텍스트 확정 후 재분석 요청 ConfirmView → confirm()
POST /problems/from-text 텍스트로 새 문제 세션 생성 SimilarProblemCard "새 대화"
Conversations
GET /conversations 전체 대화 목록 NotesPage
GET /conversations/:id 단일 대화 + 메시지 전체 ChatPage 초기 로드
PATCH /conversations/:id title / is_favorite 수정 AnalysisCard, RenameModal, NoteCard
DELETE /conversations/:id 대화 삭제 NotesPage 배치 삭제
POST /conversations/:id/messages 메시지 전송 (SSE 스트리밍 응답) ChatPage sendMessage()
POST /conversations/:id/similar-problem 유사 문제 생성 { difficulty } ChatPage handleDifficultySelect()
Folders
GET /folders 폴더 목록 NotesPage, AddToFolderSheet
POST /folders 폴더 생성 { name, color } FolderBottomSheet
PATCH /folders/:id 폴더 이름·색상 수정 FolderPage 편집
DELETE /folders/:id 폴더 삭제 (예약됨)
POST /folders/:id/conversations 대화를 폴더에 추가 AddToFolderSheet, NotesPage 배치
DELETE /folders/:id/conversations/:convId 폴더에서 대화 제거 AddToFolderSheet (체크 해제)
Users
GET /users/me 현재 사용자 정보 조회 userStore
